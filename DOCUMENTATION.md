## ТЕОРЕТИЧЕСКАЯ ЧАСТЬ

### 1. Сверточные нейронные сети (CNN)

#### 1.1 Основные компоненты

**Сверточный слой (Convolutional Layer)**

Сверточный слой - это основной строительный блок CNN. Он выполняет операцию свертки над входными данными.

Математически операция свертки описывается формулой:

```
S(i,j) = (I * K)(i,j) = ΣΣ I(i+m, j+n) * K(m,n)
```

Где:
- I - входное изображение
- K - ядро свертки (kernel/filter)
- S - выходная карта признаков (feature map)

**Принцип работы:**

1. Ядро свертки (обычно 3x3 или 5x5) "скользит" по изображению
2. На каждой позиции вычисляется скалярное произведение
3. Результаты формируют карту признаков

**Что извлекает свертка:**
- **Первые слои**: края, линии, простые текстуры
- **Средние слои**: формы, паттерны (глаза, нос, рот)
- **Глубокие слои**: сложные признаки (выражения лица)

**Batch Normalization**

Нормализация активаций между слоями:

```
y = γ * (x - μ) / √(σ² + ε) + β
```

**Преимущества:**
- Ускоряет обучение
- Стабилизирует процесс
- Действует как регуляризатор

**Pooling (Субдискретизация)**

MaxPooling выбирает максимальное значение в окне:

```
y(i,j) = max{x(i*s+m, j*s+n) | 0 ≤ m,n < k}
```

Где:
- k - размер окна
- s - шаг (stride)

**Функции активации**

ReLU (Rectified Linear Unit):
```
f(x) = max(0, x)
```

**Преимущества:**
- Простота вычислений
- Решает проблему затухающего градиента
- Обеспечивает разреженность активаций

#### 1.2 Архитектура сети

Наша модель состоит из:

```
Input (48x48x1)
    ↓
[Conv → BN → ReLU → Conv → BN → ReLU → MaxPool → Dropout] x 4
    ↓
Flatten
    ↓
[FC → BN → ReLU → Dropout] x 2
    ↓
FC → Softmax → Output (3 класса)
```

**Размеры на каждом этапе:**

1. **Блок 1**: 48x48x1 → 48x48x64 → 24x24x64
2. **Блок 2**: 24x24x64 → 24x24x128 → 12x12x128
3. **Блок 3**: 12x12x128 → 12x12x256 → 6x6x256
4. **Блок 4**: 6x6x256 → 6x6x512 → 3x3x512
5. **FC слои**: 4608 → 512 → 256 → 3

### 2. Обучение нейронной сети

#### 2.1 Функция потерь

Кросс-энтропия для многоклассовой классификации:

```
L = -Σ y_i * log(ŷ_i)
```

Где:
- y_i - истинная метка (one-hot encoded)
- ŷ_i - предсказанная вероятность класса i

#### 2.2 Оптимизация

**Adam Optimizer**

Адаптивный метод оптимизации, объединяющий:
- Momentum: использует экспоненциально взвешенное среднее градиентов
- RMSprop: адаптирует learning rate для каждого параметра

```
m_t = β₁ * m_{t-1} + (1-β₁) * g_t
v_t = β₂ * v_{t-1} + (1-β₂) * g_t²
θ_t = θ_{t-1} - α * m̂_t / (√v̂_t + ε)
```

**Параметры:**
- α = 0.001 (learning rate)
- β₁ = 0.9
- β₂ = 0.999

#### 2.3 Регуляризация

**Dropout**

Случайно отключает нейроны с вероятностью p:

```
y = x * Bernoulli(1-p) / (1-p)
```

**Используемые значения:**
- Сверточные слои: p = 0.25
- Полносвязные слои: p = 0.5

**Data Augmentation**

Искусственное увеличение датасета через трансформации:
- Горизонтальное отражение (p=0.5)
- Поворот (±10°)
- Масштабирование (0.9-1.1)
- Сдвиг (±10%)

### 3. Обработка изображений

#### 3.1 Предобработка

1. **Детекция лица**: Haar Cascade Classifier
2. **Преобразование в Grayscale**: уменьшение размерности
3. **Изменение размера**: 48x48 пикселей
4. **Нормализация**: (x - 0.5) / 0.5 → [-1, 1]

#### 3.2 Haar Cascade

Алгоритм обнаружения объектов на основе признаков Хаара:

1. Вычисление интегрального изображения
2. Применение каскада классификаторов
3. Отсеивание ложных срабатываний

## РЕЗУЛЬТАТЫ И АНАЛИЗ

### 1. Метрики качества

В версии B2 метрики зависят от размера и качества пользовательского датасета.
Скрипт `train_fer2013_b2.py` выводит лучшую точность на валидации (`best val acc`).
Для диплома рекомендуется привести фактические значения после финального обучения.

### 2. Матрица ошибок

Матрица ошибок формируется на выбранном валид‑наборе и показывает,
какие из 3 эмоций чаще всего путаются между собой.
Основные причины ошибок:
- Дисбаланс классов
- Схожесть выражений лица
- Качество изображений и разметки

### 3. Анализ сложности

**Временная сложность:**
- Обучение: O(n * m * k) где n - эпохи, m - батчи, k - параметры
- Инференс: O(1) для одного изображения (~50ms на CPU)

**Пространственная сложность:**
- Модель: ~10 MB
- Память во время обучения: ~2 GB

### 4. Сравнение с аналогами

Сравнение выполняется по критериям:
- Точность на валид‑наборе
- Время инференса
- Простота интеграции
- Объяснимость и учебная ценность
