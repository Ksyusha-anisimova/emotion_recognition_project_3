## ОТВЕТЫ НА ТИПИЧНЫЕ ВОПРОСЫ

### Технические вопросы

**Q: Почему выбрана именно такая архитектура?**
A: Архитектура основана на принципах современных CNN:
- Увеличение глубины для извлечения сложных признаков
- Batch Normalization для стабилизации обучения
- Dropout для предотвращения переобучения
- Постепенное увеличение количества фильтров (64→128→256→512)

**Q: Как работает backpropagation в CNN?**
A: 
1. Вычисление ошибки на выходе (loss)
2. Распространение градиента назад через слои
3. Обновление весов по правилу: w = w - lr * ∂L/∂w
4. Для сверток используется chain rule для градиентов

**Q: Почему точность на валидации ниже, чем на обучении?**
A: Это нормальное явление:
- Модель оптимизируется под обучающую выборку
- Валидационная выборка содержит новые примеры
- Возможен небольшой overfitting
- Используется регуляризация для минимизации разрыва

### Практические вопросы

**Q: Как можно улучшить точность?**
A: Несколько подходов:
1. Больше данных (использовать AffectNet, RAF-DB)
2. Data augmentation (более агрессивные трансформации)
3. Transfer learning (использовать предобученные веса)
4. Ensemble learning (комбинирование моделей)
5. Attention механизмы
6. Балансировка классов

**Q: Какие эмоции чаще путаются между собой?**
A: Обычно это:
1. Грусть ↔ Нейтральное
2. Счастье ↔ Нейтральное
Причины: схожесть выражений, качество данных и дисбаланс классов

**Q: Как система работает в production?**
A: Текущая реализация - proof-of-concept. Для production нужно:
1. Оптимизация модели (quantization, pruning)
2. Использование ONNX для кросс-платформенности
3. Добавление мониторинга и логирования
4. Масштабирование (Kubernetes, Docker)
5. A/B тестирование

### Теоретические вопросы

**Q: В чем отличие CNN от обычной нейронной сети?**
A: 
- **CNN**: локальные связи, разделение весов, пространственная структура
- **Обычная NN**: полносвязные слои, каждый нейрон связан со всеми
- **Преимущества CNN**: меньше параметров, инвариантность к сдвигам

**Q: Что такое receptive field?**
A: Область входного изображения, которая влияет на конкретный нейрон.
- Увеличивается с глубиной сети
- В нашей модели: от 3x3 в первом слое до ~48x48 в последнем

**Q: Зачем нужен Batch Normalization?**
A: 
1. Нормализует активации между слоями
2. Ускоряет обучение (можно использовать больший learning rate)
3. Действует как регуляризатор
4. Уменьшает internal covariate shift
