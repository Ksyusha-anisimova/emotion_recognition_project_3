# Дизайн: проверка главы 1 на корректность и соответствие проекту

## Цель
Проверить главу 1 в `диплом.docx` на:
- соответствие фактической реализации проекта (код, конфигурация, датасеты, ограничения),
- внешнюю достоверность фактов (литература, стандарты, утверждения о методах/подходах),
- логическую связность.

Результат — отчёт в чате с:
- незначительными замечаниями (без переписывания),
- значимыми несостыковками (с переписанными фрагментами),
- ссылками на внешние источники по каждому факту,
- предложением нового названия главы 1, отражающего все подпункты.

## Объём
Глава 1 (подразделы 1.1–1.9) в `диплом.docx`.

## Подход
Гибридный:
1. Извлечь текст главы 1 из DOCX и пронумеровать абзацы.
2. Сверить утверждения с проектом: код в `model/`, `web_app/`, `utils/`, и документы (`README.md`, `DOCUMENTATION.md`, `PROJECT_SUMMARY.md`).
3. Провести факт‑чек по внешним источникам для утверждений о методах, датасетах, платформах и общих фактах.
4. Сформировать отчёт:
   - Незначительные замечания — отметить без переписывания.
   - Значимые несостыковки — дать Draft rewrite → «What makes the below so obviously AI generated?» → Final rewrite.
   - Все факты и переписанные фрагменты снабдить ссылками.
5. Предложить 2–3 варианта названия главы 1, согласованные со стилем глав 2–3.

## Формат пункта отчёта
- Локация: подраздел, абзац, фрагмент
- Тип: незначительное / значимое
- Проблема
- Источник(и)
- Действие (только для значимых): Draft rewrite → AI‑audit → Final rewrite

## Ограничения
- Репозиторий без git — коммит невозможен.
- Внешняя достоверность подтверждается только по публичным источникам.
