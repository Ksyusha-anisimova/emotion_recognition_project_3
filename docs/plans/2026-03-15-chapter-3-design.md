# Design: Глава 3 (структура проекта, UML, код, интерфейс)

## Контекст
Отчет пишется по проекту системы распознавания эмоций. Требуется продолжить работу над главой 3: показать внутреннюю структуру, UML‑диаграммы, ключевые фрагменты кода и описание интерфейса. Оформление должно соответствовать стилю исходного `диплом.docx`.

## Цели
- Добавить главу 3 с разделами 3.1–3.5.
- Вставить 3 UML‑диаграммы (component, class, sequence) и места под рисунки в документ.
- Привести текст и примеры кода строго по фактической реализации в репозитории.

## Ограничения
- Не выдумывать факты, только по файлам проекта.
- Без дублирования ранее описанного материала.
- Диаграммы хранить в `report_assets/` в формате draw.io.

## Подход
1. Создать draw.io диаграммы: component, class, sequence.
2. Вставить в отчет главу 3 с разделами:
   - 3.1 Общая структура проекта
   - 3.2 Модули и их взаимодействие
   - 3.3 UML‑диаграммы
   - 3.4 Реализация ключевых идей (короткие код‑фрагменты)
   - 3.5 Интерфейс и сценарии работы
3. Добавить в документ плейсхолдеры под рисунки 3.1–3.3.
4. Все новые вставки выдержать в стиле текущего документа.
