# Дизайн: проверка главы 1.6 и список правок в отдельном DOCX

## Цель
Проверить главу 1.6 "Анализ существующих решений и их рыночных ограничений" на логику, корректность и достоверность, выполнить внешний факт‑чек, и подготовить отдельный файл `правки.docx` со списком правок и ссылками на источники.

## Объём
- Только глава 1.6 в `диплом.docx`.
- Результат: список правок (не переписанный полный текст).
- Формат ссылок на место правки: "Глава 1.6, абзац N, фрагмент ...".

## Подход
Гибридный:
1. Извлечь текст главы 1.6 для удобного чтения.
2. Сверить нумерацию абзацев с XML (`document.xml`) для точных ссылок.
3. Провести внешний факт‑чек ключевых утверждений.
4. Сформировать `правки.docx` со структурированными пунктами правок.

## Формат одной правки
- Локация: Глава 1.6, абзац N, фрагмент …
- Тип проблемы: логика / фактическая ошибка / недостоверность / отсутствие источника
- Пояснение: кратко что не так
- Источник(и): ссылки
- Предложенная правка:
  - Draft rewrite
  - "What makes the below so obviously AI generated?" (если применимо)
  - Final rewrite

## Ограничения и риски
- `pandoc` может слегка сдвигать абзацы; потому нумерация подтверждается через XML.
- Репозиторий без git, коммит выполнить нельзя.
