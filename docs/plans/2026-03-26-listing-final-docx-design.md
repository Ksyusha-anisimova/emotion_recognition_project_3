# Листинг итогового кода в .docx — дизайн

## Цель
Сформировать итоговый листинг «основных» программ проекта и сохранить его в .docx для вставки в диплом. Листинг должен быть чистым: без комментариев и докстрингов, кроме редких коротких пояснений, если без них смысл кода заметно теряется.

## Объем
Включаем только ядро приложения:
- `run.py`
- `model/*.py`
- `utils/*.py`
- `web_app/app.py`
- `web_app/templates/index.html`
- `web_app/static/css/style.css`
- `web_app/static/js/script.js`

Исключаем данные, чекпойнты, документацию, отчеты, изображения, `__pycache__`, `docs/`, `data/`, `report_assets/`, `.docx` и прочие не‑кодовые артефакты.

## Обработка кода
- Удалить все комментарии (`#`, `//`, `/*...*/`, `<!-- -->`) и docstring’и.
- Оставить только минимальные ручные пояснения, если без них теряется читабельность. Тон — нейтральный, человеческий, без «ИИ‑маркетинга».
- Не менять логику кода.

## Формат .docx
- Для каждого файла: заголовок с путем, затем моноширинный блок кода.
- Разделитель: разрыв страницы между файлами.
- Размер страницы: US Letter, поля 1".
- Шрифт: Arial; код — моноширинный (Courier New).

## Выход
Создать файл:
`/Users/admin/Desktop/универ/4-курс/диплом/листинг-итогового.docx`

## Проверки
- Визуально проверить несколько файлов на отсутствие комментариев.
- Убедиться, что список файлов соответствует согласованному составу.
