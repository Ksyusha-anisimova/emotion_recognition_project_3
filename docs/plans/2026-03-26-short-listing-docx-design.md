# Сокращенный листинг в .docx — дизайн

## Цель
Создать компактный листинг ключевых ML‑файлов проекта в формате .docx для диплома. Убрать фронтенд и второстепенные скрипты, оставить только ядро обучения/оценки.

## Объем (минимум)
- `run.py`
- `model/cnn_architecture.py`
- `model/data_loader.py`
- `model/train_fer2013_b2.py`
- `model/evaluate_confusion_fer2013_b2.py`

## Очистка кода
- Удалить комментарии и docstring’и.
- Оставить код без изменений логики.

## Формат .docx
- Заголовок файла + моноширинный код.
- Разрыв страницы между файлами.
- Страница US Letter, поля 1".
