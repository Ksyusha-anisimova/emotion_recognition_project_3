# Design: дипломный отчет (анализ, перенос, правки, единый стиль)

## Контекст
Проект: система распознавания эмоций. В корне есть исходный `диплом.docx`, который задает оформление. Требуется проанализировать проект и документ, проверить соответствие требованиям преподавателя, перенести/исправить существующие главы, затем продолжить отчет без выдуманных данных.

## Цели
- Сохранить единый стиль оформления на всем протяжении отчета.
- Перенести существующие разделы в новый документ, корректно их исправить и пометить правки цветом.
- Описывать только то, что реально есть в проекте.

## Ограничения
- Нельзя добавлять факты, которых нет в коде/документации проекта.
- Нельзя дублировать материал, избегать перефразирования ради объема.
- Исправления отмечаются цветом текста (без tracked changes).

## Подход
1. Извлечь структуру и стили из `диплом.docx` (заголовки, списки, подписи, интервалы, нумерация, шрифты, рисунки).
2. Проанализировать проектные файлы (код, README, документация) и уже написанные главы.
3. Сверить написанное с требованиями преподавателя и составить список несоответствий.
4. Создать папку `report_assets/` для схем в формате draw.io.
5. Создать новый файл `отчет_диплом.docx` со стилем, идентичным исходному документу.
6. Перенести главы в новый документ. Внести правки: корректировки выделять цветом и кратко пояснять причину рядом.
7. Подготовить план продолжения текста (гл. 1–4) и список необходимых схем/рисунков.

## Выходные артефакты
- `report_assets/` с заготовками `.drawio`.
- `отчет_диплом.docx` с сохранением оформления и пометками правок.

## Риски
- Потеря стилей при переносе: минимизируется работой напрямую с XML исходного DOCX.
- Добавление недостоверных сведений: устраняется опорой на фактические файлы проекта.

## Проверки качества
- Сравнение стилей заголовков/списков/подписей с исходным `диплом.docx`.
- Ручная сверка: нет ли фактических ошибок или выдуманных данных.
- Проверка: все правки помечены цветом и краткой причиной.
