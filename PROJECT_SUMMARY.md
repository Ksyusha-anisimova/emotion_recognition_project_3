## 🎯 ФУНКЦИОНАЛЬНОСТЬ

### Веб-приложение

1. **Загрузка изображений**
   - Drag & Drop интерфейс
   - Поддержка PNG, JPG, JPEG
   - Автоматическая детекция лица

2. **Работа с веб-камерой**
   - Real-time захват
   - Мгновенный анализ
   - FPS ~20

3. **Визуализация результатов**
   - Определенная эмоция с emoji
   - Уровень уверенности (0-100%)
   - График всех вероятностей
   - Подсветка лица на изображении

4. **Информационная страница**
   - Описание проекта
   - Архитектура модели
   - Список эмоций
   - Технологии

### Модель машинного обучения

**Архитектура CNN (B2):**
```
Input (48×48×1)
    ↓
4x [Conv-BN-ReLU-Conv-BN-ReLU-Pool-Dropout]
    ↓
Flatten → FC(512) → FC(256) → FC(3)
    ↓
Output: 3 эмоции
```

**Распознаваемые эмоции:**
1. 😊 Счастье (Happy)
2. 😢 Грусть (Sad)
3. 😐 Нейтральное (Neutral)

## 🚀 КАК ЗАПУСТИТЬ

### Вариант 1: Быстрая демонстрация

```bash
cd emotion_recognition_project
pip install -r requirements.txt
python run.py demo
```

Откройте браузер: **http://localhost:5000**

### Вариант 2: С обучением модели (B2)

```bash
# Установка зависимостей
pip install -r requirements.txt

# Обучение модели
python run.py train --epochs 30 --train-dir data/custom_train_b2

# Запуск приложения
python run.py run
```

### Вариант 3: Ручной запуск

```bash
# Обучение (B2)
cd model
python train_fer2013_b2.py --train_dir ../data/custom_train_b2 --epochs 30

# Запуск веб-приложения
cd ../web_app
python app.py
```

## 📊 ОЖИДАЕМЫЕ РЕЗУЛЬТАТЫ

### Метрики качества

В версии B2 метрики зависят от размера и качества пользовательского датасета.
Скрипт обучения выводит `best val acc` после каждой эпохи.
Для диплома рекомендуется зафиксировать фактические значения после финального обучения.

### Производительность

- **Время обучения:** ~2 часа (50 эпох на GPU)
- **Inference время:** ~50ms на изображение (CPU)
- **FPS веб-камеры:** ~20
- **Размер модели:** ~10 MB
